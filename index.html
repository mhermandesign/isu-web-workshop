What Is Web Design?

Web design is the process of using design elements and principles to deliver a visually appealing message.

Be it a simple, single page brochure site for a local restaurant, or a feature heavy e-commerce site for an international company, the same considerations that go into print design apply to creating content for the web.

When you get down to the nitty-gritty, aside from some technical differences, the same skills apply to both sides of the design coin. Just like with print, it's up to the designer to make the most of the tools available.

HTML

Nearly every website you come across will be built in a standardized language called HTML (Hyper Text Markup Language).

A website will often feature many different elements including, but not limited to...

Tags

Element names surrounded by angle brackets.

Head

The section of an HTML document that holds background information like page titles, internal links, and meta-data read by the browser.

Body

The section of an HTML document that holds the actual content of a website. This includes things like images, paragraphs, videos.

Div

An empty element free of default styles.

Browser

Web browsers (Chrome, I.E., Firefox, Safari, etc.) are used to read and render HTML documents.

Text Editor

Used, in part, to create and modify HTML and CSS documents. (Notepad - PC / TextEdit - Mac)

CSS

Once a web designer finishes creating the markup for their website, they move onto creating what is called a cascading stylesheet. These stylesheets are a standardized language used to give HTML elements different visual properties like font, color, size, and orientation.

Some of the more basic terms you will come across when working with CSS are...

Rule Set

A combination of a selector and a declaration block.

Declaration Block

A combination of a style property and its associated value.

Selector

Used to "find" HTML elements based on their name, id, class, attribute, etc.

ID

A name given to an element of a page that is to be unique to that page.

Class

An attribute that can be given to any HTML element as a way to provide a style rule to multiple elements sharing that same attribute.

Basic HTML Layout

A basic HTML document will usually contain the following elements:

<html></html>

The main container of every website.

<head></head>

The brains of the operation.

<body></body>

The real meat of the website.

<div></div>

Basically as empty and devoid of personality as a bucket of rocks. At least, until you give it meaning :)

<h1></h1>

Headings 1 through 6.

<p></p>

Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

Note: Aside from a very small number of exceptions, tags must always have a closing </tag>. This helps to ensure your page renders in the browser properly. Missing closing tags can result in any number of issues so, when in doubt, just add one.

Using the tags listed in the section above, the following layout can be produced:

<!doctype html> <!-- This MUST BE the very first tag in your HTML document. -->

<html>


<head>

<title>My Website</title>

</head>


<body>

<div id="blah">

<h1>Hello, World.</h1>

<p>Lorem ipsum dolor sit.</p>

</div>

</body>


</html>

The above markup will produce something similar to the following:

Hello, World.

Lorem ipsum dolor sit.

Basic CSS Implementation

Some of the more common, basic examples of CSS descriptors include:

color: black | #000000 | rgba(0,0,0,1);

background-color: blue | #0000FF | rgba(0,0,255,1);

border: solid 3px blue;

font-size: 21px | 1em | 15%;

width: 100px | 50% | 10vw;

text-align: center | left | right;

margin: 20px | 1vw | 10%; /*A margin is the space between the outside edge of an element and those surrounding it.*/

padding: 20px | 1vw | 10%; /*A padding is the space between the inside edge of an element and those inside of the element itself.*/

Note: Similar to the way HTML tags must have a closing </tag>, CSS declaration blocks must always end with a semi-colon ( ; ) to ensure that the document is parsed and rendered correctly.

The simplest way to implement CSS styles is to include them directly inside an element within the HTML document.

<!doctype html>

<html>


<head>

<title>My Website</title>

</head>


<body style="font-family: sans-serif; height: 200px; background-color: cornflowerblue">

<div id="blah">

<h1 style="color: white;">Hello, World.</h1>

<p style="font-weight: bold;">Lorem ipsum dolor sit.</p>

</div>

</body>


</html>

The above markup will produce something similar to the following:

Hello, World.

Lorem ipsum dolor sit.

While the previous example technically works, it's nearly unreadable. To remedy this, designers use stylesheets which look something like this:

body {

font-family: sans-serif;

background-color: cornflowerblue;

height 200px;

}


h1 {

color: white;

}


p {

font-weight: bold;

}
